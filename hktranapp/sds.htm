使用 Swift Actor 實現的 KMBRateLimiter，確保每次請求間隔不少於 500ms。多個組合共享同一車站時，會合併為單一 API 呼叫。

5. 服務層設計

5.1 ETA 刷新引擎（ETARefreshEngine）

參數值
自動刷新間隔30 秒
API 超時10 秒
MTR/CTB 並發完全並行
KMB 並發按 stopId 批次（500ms 間隔）
離線行為跳過刷新，保留快取

5.2 網絡監控（NetworkMonitor）

  • 基於 NWPathMonitor 的單例
  • 狀態：上線（online）、離線（offline）、未知（unknown）
  • 網絡恢復時自動觸發 ETA 刷新

5.3 路線搜尋引擎（RouteSearchEngine）

  • 從 SwiftData 載入所有 RouteIndexEntry（~800-900 條）
  • 記憶體內篩選：路線號碼 → 前綴 → 關鍵字
  • 排序：交通工具優先級 → 精確匹配 → 前綴匹配
  • 搜尋記錄：UserDefaults，最多 10 條

6. ETA 狀態機

每張路線卡片根據快取新鮮度及網絡狀態，顯示 7 種視覺狀態：

狀態觸發條件視覺效果
正常（Fresh）快取 < 90 秒正常透明度
稍舊（Stale）90 秒 – 5 分鐘80% 透明度，琥珀時鐘
離線（Offline）5–30 分鐘或無網絡60% 透明度，離線圖示
錯誤（Error）> 30 分鐘或 API 失敗紅色三角形
暫停服務API 回傳空陣列月亮圖示 +「暫停服務」
受限（Degraded）ETA > 120 分鐘「點擊查看站名」
載入中首次載入，無快取載入動畫

7. 離線與錯誤處理

  • API 失敗不清除快取：用戶始終看到最後已知數據
  • 逐個交通工具隔離：KMB 故障不影響 MTR/CTB 數據
  • 離線模式：顯示快取資料 + 離線提示橫幅
  • 網絡恢復：自動觸發即時刷新
  • 首次啟動離線：顯示專用恢復頁面 + 重試按鈕

4.2 KMB Rate Limiter

Implemented as a Swift Actor (KMBRateLimiter), ensuring ≥500ms between API requests. Multiple combinations sharing the same stop are batched into a single API call.

5. Service Layer Design

5.1 ETA Refresh Engine

ParameterValue
Auto-refresh interval30 seconds
API timeout10 seconds
MTR/CTB concurrencyFull parallel
KMB concurrencyBatched by stopId (500ms spacing)
Offline behaviorSkip refresh, preserve cache

5.2 Network Monitor

  • Singleton based on NWPathMonitor
  • States: online, offline, unknown
  • Auto-triggers ETA refresh on network restore

5.3 Route Search Engine

  • Loads all RouteIndexEntry records from SwiftData (~800-900 entries)
  • In-memory filtering: route number → prefix → keyword
  • Sorting: operator priority → exact match → prefix match
  • Search history: UserDefaults, max 10 items

7. Offline & Error Handling

  • API failures never clear cache: Users always see last known data
  • Per-operator isolation: KMB outage doesn't affect MTR/CTB data
  • Offline mode: Cached data + offline banner indicator
  • Network restore: Auto-triggers immediate refresh
  • First launch offline: Dedicated recovery view + retry button
